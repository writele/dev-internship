Design Questions

  1. Explain how design and development intersect. What role do you see a developer playing in the design process?

    2. Design is an important component of any website. It's easy for design and development to blend, as the front-end developer is implementing the design. There are going to be considerations that are better understood from a design standpoint than from a development perspective, and vice versa.

    The role of the developer in the design process is likely going to depend on the needs of the team and project. For smaller teams, it probably makes sense for the developer and designer to collaborate more so than in large companies with separate departments.

    From my perspective, the developer's main obligation is to give feedback on the feasibility of a design. If aspects of a design are going to cause problems in adhering to code standards and/or cross-browser compatibility, the developer should definitely mention this and suggest alternatives.

    Design isn't always a linear process, in which the design is completely created up front and then the developer implements it. I can see how the design could end up changing due to communication between the designer and developer.

    Ultimately, I think most final decision-making regarding the design should be left up to the designer. How much feedback the developer should give depends on the nature of the team and project.

  2. Describe how grids are useful, and how you generally go about using them, or implementing them.

    3. Grids provide a reusable CSS structure for implementing the layout of elements on a webpage. In a grid system, the page is divided into columns contained in rows. By attaching a column class to an html element (usually a div), you can specify how much that element spans across the page. For instance, in a 12-column grid, a class such as "12-col" would indicate that a div should span across the entire row. The initial row would span across the entire page or across a set width. A class of "4-col" would indicate that the div should span across 1/3 of the row.

    Rows can be nested within rows to provide further containers for columns. For example, you could have a row with two 6-column divs. If you want to evenly display two content boxes within one of those divs, you would add another row with two 6-column divs.

    Grids make positioning elements on a page fast and easy. Additionally, grids are great for responsive design. By utilizing percentages on columns, the columns will grow larger or smaller depending on screen size.

  3. Talk a little bit about pattern libraries and visual systems. What are they and how are they useful? Have you used them before?

    4. A pattern library is a collection of design patterns. These tend to include documentation for page elements such as navigation, buttons, the grid layout, tables, lists, and forms. A pattern library may contain code snippets and accompanying CSS files. Sometimes simple Javascript widgets, such as a drop-down menu or accordion, are also featured.

    These patterns can be used to create consistent user experience throughout a large website. Having reusable patterns that adhere to certain code standards makes websites easier to maintain. Creating a pattern library can allow you to test your code up front, to ensure that it works across browsers and meets accessibility standards. It also prevents members of large teams from recreating something that's already been done, which cuts down on costs.

    The closest I've gotten to using pattern libraries is through utilizing front-end frameworks such as Bootstrap and Foundation. These aren't exactly pattern libraries, since they were designed for general use rather than to meet the specific needs of a particular website. I would look forward to the opportunity to use and contribute to a pattern library, as they look like great tools.

  4. What is your comfort level with responsive design? Have you done any mobile-first projects before? Tell us about what you know.

    5. I try to keep responsive design in mind for every project. At its simplest, I've made sure my websites used percentage-based layouts. For most projects, I've implemented media queries and break points so that the content spread out at larger screen sizes, but every div was contained to its own row on mobile devices.

    Images are another important consideration in responsive design. Displaying the same large images for all screen sizes adds unnecessarily to load time. For A Tribute to Phoebe Snow, my client provided several images they wanted featured. I used srcset and PictureFill.js so that the images displayed would differ depending on the screen size.

Even though I ended up picking out a theme to use, keeping mobile-first in mind allowed me to focus on the most essential content for the main page. Although I have not always designed this way, I understand the benefits of mobile-first design. Namely, it allows you to focus on what is most important before enhancing functionality.

  7. Explain how HTML, CSS, and JS interact - what is each layer responsible for?

    8. HTML provides content and structure to a webpage. Ideally, it allows everyone, including older browsers and speech-readers, to easily access your content and navigate through your website. This is the first layer required for a website to provide functionality.

    CSS is the next layer, which allows you to add visual style to a webpage.

    Finally, Javascript adds in a layer of behavior, so that, provided Javascript is enabled, elements on the page can change depending on the user input.

  8. Progressive enhancement or graceful degredation? Choose your side.

    9. Progressive enhancement and graceful degradation are two different methods of ensuring your websites are accessible to all users. In my opinion, progressive enhancement is the better strategy because it provides a better user experience and makes projects easier to maintain in the long run.

    Through progressive enhancement, websites are built with a strong foundation up-front that focuses on content rather than presentation. Graceful degradation focuses on complex presentation that makes allowances for users with older technology. A feature that requires Javascript might present an error message for users who have Javascript disabled. In a website built on progressive enhancement, however, the option would not be shown at all to users with Javascript, nor would the option be necessary to provide a positive user experience.

    The advantage of progressive enhancement is that in the initial product, you have already provided a website that will be pleasant for all users. Newer browsers may have additional features, but this will be "icing on the cake", so to speak. Although this requires more planning up front, there won't be any need to alter the original code when new technologies are released. It also ensures that there is one project that is kept up to date, rather than different versions of the same product built for various scenarios. With progressive enhancement, a team can "release early, release often" and focus on building enhancements instead of fixing edge cases.
